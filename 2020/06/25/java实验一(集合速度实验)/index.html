    结果讨论

    对于增删

    首先请看Arraylist源码

    再看LinkedList源码

    通过Arraylist和linkedList源码可知道，add的两个重载函数是不一样的;

    明显 add() 比add(index,element)速度快，后者比前者多了两个步骤：判断数组下标是否越界；将添加位置后面的数据进行copy。

    所以即便是在尾部插入一个数据，这两种方法都可以实现，但是明显add()方法比较快。

    结果

    对于不同类之间的比较速度：

    1、当在接近头部的位置插入时，linkedList速度是一定大于Arraylist的；

    2、在尾部插入时候数据量小的时候差别不大，当达到百万级别的时候有的时候ArrayList快有的时候LinkedList快，但是平均下来还是ArrayList的快。

    原因

    1、因为LinkedList插入是先通过for循环遍历找到index然后直接插入，而ArrayList在指定位置添加或者删除元素主要耗时间的地方在与拷贝index后面元素，（这个操作会移动index后面所有的元素），所以在接近头部的位置插入LinkedList遍历很快找到index的位置，但是Arraylist的复制数组会耗费很多时间。

    2、思考： 为什么在尾部插入速度比较不出来？

    虽然本次实验平均过百万级别的速度是ArrayLsist快，但是我们仍然不能说明是哪个比较快。

    对于遍历

    毋庸置疑的说是选用Arraylist更快一点，因为通过源码我们可以知道get方法是直接返回index位置上的元素

    而Linkedlist需要先遍历节点。
